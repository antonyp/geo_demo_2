Courier Bookings

This demonstrator shows how a courier booking system could work and how it could be visualised on a map using Google Maps.

  1. A booking is entered into the system with Address, Label, Date and Time for collection.
  2. The address is geocoded using the Google Geocoding API so it can be ploted on the map geospatially.
  3. A geospatial operation is performed with this point against courier zones to assign the booking to a courier.
  4. At the start of the cycle in which the parcel should be collected the allocated courier is automatically notified of the collection job.
  5. The track and trace data is monitored for real time exception reporting, in real time on the map Customer Service staff have the ability to see all bookings that are overdue, the courier is notified that a collection has been missed. The customer service staff are also able to search for jobs by courier, by address and by label number in order to ascertain current status.
